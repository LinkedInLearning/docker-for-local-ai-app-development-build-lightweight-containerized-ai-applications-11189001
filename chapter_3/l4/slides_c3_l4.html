The tight loop

No gap between writing and running.

The interpreter, the libraries, the database — everything the code needs is already here. You import a library and it's the pinned version. You connect to ChromaDB and it's right there on the network.

You're not maintaining a local environment and a container environment. There's only one — and it's the one that ships.

AI assistants in the loop

The assistant runs in the same workspace.

Tools like Claude Code see the same files your code does. They're excellent at the mechanical parts:

Scaffold a module
Draft a function
Generate tests
Explain code
You bring the architecture; the assistant accelerates the typing.

Practice · develop in modules

Small importable pieces, not one giant script.

Our RAG code under rag/ is split by responsibility — each stage its own module.

1
ingestion/pdf_parser.py Read a PDF into structured elements.
2
ingestion/chunker.py Split elements into retrievable chunks.
3
ingestion/embedder.py Turn chunks into embedding vectors.
4
store.py Write and read vectors in ChromaDB.
5
retrieval/chain.py Retrieve context and answer a query.

Practice · validate as you go

Two tools to exercise the work.

One for stepping through a pipeline, one for driving the whole system like a user.

Notebooks

Step through a pipeline cell by cell. Inspect the output of each stage and tweak a parameter before wiring stages together.

Dashboard

Exercise the whole system end to end — upload, ask, answer — standing in for a real user before anything is productionized.

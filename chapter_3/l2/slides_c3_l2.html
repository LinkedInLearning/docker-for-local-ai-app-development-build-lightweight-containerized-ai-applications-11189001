Why Compose

One docker run per container doesn't scale.

Every container we started took its own command with its own flags — ports, volumes, env vars, networks. Fine for one. It falls apart the moment two or three need to start together, talk to each other, and share config.

Compose lets you describe the whole environment declaratively in a single docker-compose.yaml, and bring it all up with one command.

Anatomy

A file describes a set of services.

docker-compose.yaml
services:
  python:
    image: rkrispin/python-dev-rag-docker:0.0.3
    # volumes, command, environment…
  chromadb:
    image: chromadb/chroma:1.3.5
    ports: ["8000:8000"]
Key → meaning
servicesEach entry is one container.
imageWhich image backs the service.
portsPublish to the host — like -p.

The shape mirrors docker run — every flag becomes a line. Now it's version-controlled, reviewable, reproducible.

The pattern

Build your app. Pull your infrastructure.

A common, important rule — and Compose is what stitches the two kinds together.

You build

Your application code

  • The dev container
  • Your API / service images
  • Anything with your logic
docker build
+
Someone else builds

Infrastructure components

  • Databases (ChromaDB, Postgres)
  • Queues, caches
  • Pinned, known-good versions
docker pull

Networking

Each service is reachable by its name.

network: rag-docker (bridge)
python
our code runs here
chromadb:8000
chromadb
8000 → published to host
Inside the network, the service name is the address: chromadb:8000. From the host, the published port is localhost:8000. depends_on makes python start after chromadb.

Hands-on · terminal

Let's run it for real.

Two services, a shared network, service-name networking — all declared in one file. We're in the chapter_3/l2 folder. Switch to the terminal.

docker compose up -d docker compose ps docker compose down

Terminal · bring it up

One command starts everything.

$ docker compose up -d
[+] Running 3/3
✓ Network l2_rag-docker Created
✓ Container l2-chromadb-1 Started
✓ Container l2-python-1 Started
$ docker compose ps
NAME SERVICE STATUS PORTS
l2-python-1 python Up
l2-chromadb-1 chromadb Up 0.0.0.0:8000->8000/tcp

Terminal · prove it works

From the host, then from inside.

$ curl http://localhost:8000/api/v2/heartbeat
{"nanosecond heartbeat": 1748390400000000000}
$ docker compose exec python bash
root@python:/workspace# curl http://chromadb:8000/api/v2/heartbeat
{"nanosecond heartbeat": 1748390400500000000}
# hostname is "chromadb", not localhost — service-name networking
$ docker compose down # stops + removes both containers and the network

Takeaway

One file. One command up, one command down.

Compose turns a multi-container environment into something reproducible and shareable.

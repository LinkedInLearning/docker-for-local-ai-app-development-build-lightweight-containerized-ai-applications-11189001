Definition

Your editor's UI on the host, its brains in the container.

VS Code keeps its familiar window on your machine. The server side — language server, terminal, extensions — runs inside the container.

VS Code UI

Inside the container

Full isolation — what you see is what runs.

One world, not two.

Without a dev container the editor and the runtime drift apart. With one, they are the same environment.

Config · 1 of 3

Build on what you have

Reuse the Compose file.

Instead of defining a new environment, devcontainer.json points at the same docker-compose.yaml and picks the python service.

A dev container can attach to a Compose service or to a plain image — either works.
.devcontainer/devcontainer.json
"dockerComposeFile": ["../docker-compose.yaml"],
"service": "python",
"workspaceFolder": "/workspace/"

VS Code starts the python service (and chromadb alongside it) and opens /workspace/ inside it.

Config · 2 of 3

Project-level extensions

The toolchain travels with the project.

List the extensions the project needs. They install inside the container, scoped to this project — not in your global VS Code.

Every teammate opens the project and gets the same Python, Ruff, and Jupyter tooling.
customizations · vscode
"customizations": {
  "vscode": {
    "extensions": [
      "ms-python.python",
      "charliermarsh.ruff",
      "ms-toolsai.jupyter"
    ]
  }
}

Config · 3 of 3

Mounts & ports

Reach beyond the project, open ports for free.

mounts binds host files from outside the project — a cache, credentials, shell history — without copying them into the image.

forwardPorts auto-publishes app ports: a Streamlit app on 8501 just opens on the host.
mounts + forwardPorts
"mounts": [
  "source=${localEnv:HOME}/.zsh_history_dev,
   target=/root/.zsh_history,type=bind"
],
"forwardPorts": [8501, 8080]

Hands-on · verify

Confirm you're really inside.

The integrated terminal runs in the container, not on your Mac. Check the interpreter.

$ which python
/opt/python-3.11-dev/bin/python
$ python --version
Python 3.11.9
$ # the interpreter baked into our image — the version we pinned

And the extensions panel shows Python, Ruff, and Jupyter all marked installed in the container — the editor's intelligence is now backed by the container's environment.

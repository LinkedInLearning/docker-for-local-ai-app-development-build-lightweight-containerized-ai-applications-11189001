A RAG system enables a language model to answer questions about information it was not trained on — by retrieving the right snippets from your documents and grounding the answer in them.

The data is converted into embeddings — numerical vectors — so the model can retrieve relevant pieces and use them when generating an answer.

Anatomy

A RAG system is two pipelines.

One prepares the knowledge base; the other answers questions using it. Each runs at a different cadence and on different triggers.

Pipeline 1

Data ingestion pipeline

documents→ parse→ chunk→ embed→ store

Runs when new documents arrive. Turns raw documents into vectors stored in a vector database.

offline · batch
Pipeline 2

Query pipeline

question→ embed→ search→ LLM→ answer

Runs on every user request. Retrieves context from the vector store, then asks the LLM to answer.

online · real-time

The two pipelines share one vector store. We'll see why that matters for container strategy in the next lesson.

Pipeline 1 · ingestion

Documents in. Embeddings out.

LLMs don't work on raw text — they work on numerical vectors. The ingestion pipeline turns documents into vectors so the model can search them later.

Each step is its own concern — parser, chunker, embedder, store. That separation will matter when we package services.

Pipeline 2 · query

Question in. Grounded answer out.

On every request: embed the question with the same model, search the vector store, optionally rerank, build a prompt, and let the LLM answer.

Beyond RAG

The same shape shows up everywhere.

The specific components change, but most modern AI applications follow a similar pattern: a batch component prepares state, an online component uses it.

AI AgentsTool definitions + state → agent loop. Same offline/online split.
Forecasting systemsTrain pipeline + scoring API. Two services, one model registry.
Recommendation systemsFeature pipeline + serving layer. Trained offline, served online.
+ logging, monitoring, UIProduction AI apps add observability and a frontend — more services, same pattern.

Takeaway

Modern AI apps are multi-component.

The takeaway isn't how RAG works internally — it's that real AI systems are several independent services working together.

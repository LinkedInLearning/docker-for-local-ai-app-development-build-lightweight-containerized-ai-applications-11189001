How the services talk

The database is the contract.

network: compose bridge
ingestion
parses + embeds documents
chromadb:8000
writes vectors →
chromadb
shared vector store
chromadb:8000
← reads vectors
query
retrieves + answers
The two services never call each other — they share state through the DB. Both reach it by service name chromadb:8000 (Compose DNS). No IP addresses.

Health & readiness

Every service answers /health.

One simple signal, used three ways — and it's the same signal a load balancer or orchestrator uses in production.

GET /health

Gate startup

The database's healthcheck holds dependents back until it's truly ready — the depends_on condition we wired in Lesson 3.

at boot
GET /health

Confirm readiness

Hit each service directly to confirm it's up before sending real traffic — exactly what an orchestrator probes to decide whether to route to a container.

before traffic

The integration test that matters

One flow only multi-container can run.

1

POST /ingest

ingestion service
2

poll GET /ingest/jobs/{id} until completed

ingestion service
3

POST /query → assert the answer has sources

query service

Passing = networking + the shared-DB contract + full integration, all proven at once.

Hands-on · terminal

Let's run it for real.

The stack from Lesson 3 is up. We'll ingest through one service, query through another, then prove service-name DNS straight from inside a container. Switch to the terminal.

curl /ingest curl /query pytest -v

Terminal · ingest then query

Two containers, one coherent result.

$ curl -X POST localhost:8081/ingest \
  -H "X-API-Key: dev-key" -d '{"source_dir":"pdf/"}'
{"job_id": "a1b2c3d4", "status": "pending"}
$ curl -X POST localhost:8080/query \
  -H "X-API-Key: dev-key" -d '{"question":"What is this document about?"}'
{"answer": "This document covers...", "sources": ["pdf/…"]}

Terminal · prove the DNS

Reach the DB by name, from inside.

$ docker compose -f chapter_4/l3/docker-compose.test.yaml \
  exec query python -c "…urlopen('http://chromadb:8000/api/v2/heartbeat').status"
200
# hostname is "chromadb", not localhost —
# service-name DNS, resolved on the Compose bridge network

Split the dependencies

Heavy vs. lean — the split pays off here.

requirements-ingestion.txt
ingestion
Parses PDFs · heavy
  • docling
  • sentence-transformers
  • langchain-text-splitters
  • chromadb
requirements-query.txt
query
Retrieve & call LLM · lean
  • fastapi · uvicorn
  • langchain-openai / -anthropic / -google
  • chromadb
  • docling

The single biggest difference: Docling is absent from the query image. The vision and parsing stack it pulls in is exactly the weight the query service never needs.

Two entry points

Each image serves only half the routes.

We split the routes into two routers and two tiny app modules. query_app.py mounts only the query router; ingestion_app.py is its mirror.

# rag/api/query_app.py
from fastapi import FastAPI
from rag.api.routes import query
app = FastAPI(title="RAG Query")
app.include_router(query.router)

The query app never imports the ingestion module, so it never imports Docling. The lean image is lean because the code says so.

The two Dockerfiles

Same best practices — three lines differ.

Dockerfile_Query · lean
FROM python:3.11-slim
WORKDIR /app
COPY requirements-query.txt requirements.txt
RUN pip install -r requirements.txt
COPY rag/ /app/rag/
EXPOSE 8080
CMD ["uvicorn","rag.api.query_app:app","--port","8080"]
Dockerfile_Ingestion · heavy
FROM python:3.11-slim
RUN apt-get install -y libgl1 libglib2.0-0
COPY requirements-ingestion.txt requirements.txt
RUN pip install -r requirements.txt
COPY rag/ /app/rag/
EXPOSE 8081
CMD ["uvicorn","rag.api.ingestion_app:app","--port","8081"]

Hands-on · terminal

Let's build both images.

We're at the project root. One docker build per Dockerfile, then compare the sizes side by side.

build rag-query build rag-ingestion docker images

Terminal · lean image

First the query image — fast.

$ docker build -f chapter_4/l2/Dockerfile_Query -t rag-query:0.1.0 .
=> [internal] load build definition from Dockerfile_Query
=> [2/4] RUN pip install --no-cache-dir -r requirements.txt
=> [3/4] COPY rag/ /app/rag/
=> naming to docker.io/library/rag-query:0.1.0
# no Docling, no system graphics libs — builds in seconds

Terminal · heavy image

Now the ingestion image — slower.

$ docker build -f chapter_4/l2/Dockerfile_Ingestion -t rag-ingestion:0.1.0 .
=> [2/5] RUN apt-get install -y libgl1 libglib2.0-0
=> [3/5] RUN pip install -r requirements.txt # docling, torch…
=> => downloading sentence-transformers, docling ~2 min
=> naming to docker.io/library/rag-ingestion:0.1.0
# the parsing stack is what makes this one take a while

Terminal · the payoff

The lean image is a fraction of the size.

# faster pulls, faster starts, smaller attack surface

Takeaway

Each service, right-sized.

One new idea — split one app into two entry points — and every image carries only what its job needs.
